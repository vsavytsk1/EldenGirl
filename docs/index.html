How this gets built

Every idea starts here as pure text and clicks — one HTML file, no dependencies, no build step, and it writes zero bytes to your device. We test the concept and the path in the community before a single UI dopamine layer goes on top, and long before anything native.

Only once a concept holds does it earn the goblin layers — the juice, the creature, the game feel — and only then is it wrapped in the secure, disguised app. Text first. Delight second. Encryption around all of it.

Why a lab, and no branches

This is a monolith on purpose. No branches, no release trains for the concepts: each one is just text and clicks that anyone can open, read, and judge. That is the whole point — a survivor’s technical friend should be able to verify every line, and a concept nobody can read is a concept nobody can trust.

writes0 bytes — no storage, no cookies, no service worker
networknone — works fully offline, nothing phones home
installnone — no manifest, leaves no trace, handed over as a URL
depszero — every file is self-contained HTML + inline JS

The rest of the project

The concept lab is the front door. Behind it, three tiers share a threat model and nothing else — the reading material here, an encrypted record on Android, and a read-only codex on iOS. Every promise is compiled into gates that fail the build rather than warn.

threat model first local-only, no server disguised by default quick exit everywhere no location, ever no analytics open source · MIT
