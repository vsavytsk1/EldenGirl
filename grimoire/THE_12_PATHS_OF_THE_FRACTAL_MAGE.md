## PATH V -- THE LANGUAGE SEAM
### curses: 1 (Curly Brace), 4 (f-string Nesting), 23 (Python Leak)

> The boundary between two worlds is always where the curse lives. It is invisible
> until the click.

When a Python script generates JavaScript, the seam between the two languages is
a single string, and inside that string the two worlds are indistinguishable. A
`{}` meant for JS collides with an f-string. A `chr(46)` meant as a dot ships raw
into JS as `ReferenceError: chr is not defined`. The bug is silent at build; it
waits for the click.

The general law (as above, so below): know exactly where one world ends and the
next begins. Build the JS as a plain string, inject once. Never write Python
builtins into emitted JS. Keep non-ASCII in DATA, never in builder source. The
seam is the killer -- so mark the seam and cross it on purpose, never by accident.

THE PRICE: the discipline to keep the two worlds separate in your head. Cheap to
pay up front; brutal to debug if you don't.

## PATH VI -- ONE SCRIPT, ONE RUN
### curses: 14 (gitiumCurse), 16 (shadowInject), 17 (regexDevour), 19 (shellDevour)

> Batch everything. Normalize, then patch. Run once. Verify. git restore is the exorcism.

Incremental patches on a Windows HTML file accumulate carriage returns until the
file is garbage (Curse 14). Greedy regex eats 133KB of a file in one bite (Curse
17). Inject into an already-injected base and you get two copies fighting (Curse
16). After ~44 tool calls the shell itself starts swallowing output (Curse 19).

The single counter-hex for all of them: **write ONE clean script that does ALL
the changes. Normalize line endings first (`\r\n -> \n`). Patch on the clean
string. Write UTF-8, `newline="\n"`, no BOM. Run once. Verify loneCR==0. Commit.**
If it breaks, `git checkout` the last clean commit -- do not cry, just restore.
If you need 44 tool calls to do one thing, you are doing it wrong: it is a design
signal to batch.

THE PRICE: the up-front thinking to design the one script. The mage who patches
incrementally pays in rotted files and lost hours -- and if they push the rot,
the reader pays too.

## PATH VII -- THE LAZY MACHINE & THE WEDGED HOST
### curses: 18 (windowsDevour), 28 (hostWedge), 29 (deployLag)

> The tools lie by omission. Print the host. Wait for green. git push is not deploy.

The machine is not malicious. It is lazy, and it lies by silence:
- Curse 18 -- `.py` opens Notepad instead of running; nothing tells you. Use the
  full interpreter path (`py -3`).
- Curse 28 -- a wedged PSES console returns bare `^C` forever while a sibling
  terminal works fine. Print `$Host.Name`; if it is the "Visual Studio Code Host"
  and silent, abandon it for a plain `pwsh`. (A command can even RUN while showing
  `^C` -- the output is eaten, not the command. Always re-verify on-disk state.)
- Curse 29 -- `git push` returns in one second, but GitHub Pages runs a separate
  deploy job (~30-90s). Check the URL immediately and you get a false 404. Watch
  the deployment go green, THEN verify.

The meta-lesson: git + Windows + the browser are all the same kind of lazy, each
with its own clock. The fast agent must slow to the pipeline's pace. Do not spam,
do not poll, do not sleep-hammer. End the beat, let the machine breathe, come back.

THE PRICE: patience. The impatient mage debugs ghosts and blames clean code.

## PATH XI -- THE CLONE MIRAGE (KNOW WHAT YOU ARE)
### curse: 27 (originMirage) . cousin of curse 6 (File:// Lie)

> The folder's name is a mirage. Its origin is the truth. One project, one clone.

A folder named JARVIS whose `git remote` points at a different repo entirely; a
`.git`-less copy that looks identical but has no history; three folders that are

fix: a thing's identity is `git remote get-url origin`, never its name on disk.
Sweep every clone for name-vs-origin mismatch, divergence, and history-less twins.

shared origin to "tidy up."

The soul-mirror: know what you actually are, not what you are labeled. Do not let
the name on the door lie about the origin inside. And when you find a marooned

THE PRICE: the paranoia sweep, and the honesty to confirm before deleting. The
